Enhanced Label‑Conditioned Schrödinger Bridge
with Ornstein‑Uhlenbeck Reference (mvOU‑SBP)

This project implements a generative model based on the Schrödinger Bridge problem. It learns a stochastic process that transports a simple prior (standard Gaussian) to a complex data distribution over a fixed time interval, while being explicitly conditioned on class labels. The model combines a conditional Variational Autoencoder (VAE) for latent representation with a drift network that learns the bridge dynamics. An optional Ornstein‑Uhlenbeck (OU) reference process provides a theoretically grounded prior for the bridge (mvOU‑SBP).

1. Mathematical Foundations

1.1 Schrödinger Bridge Problem

The Schrödinger Bridge seeks the most probable evolution between two probability distributions over a fixed time interval \([0,1]\). Given a reference process (e.g., Wiener or Ornstein‑Uhlenbeck), we look for a process whose marginal distributions at \(t=0\) and \(t=1\) match given target distributions, while minimizing the Kullback–Leibler divergence to the reference.

In generative modelling we set:

  • \(t=0\): Prior distribution \(P_0 = \mathcal{N}(0, I)\) in latent space.
  • \(t=1\): Data distribution \(P_1 = q_\phi(z|x,y)\) (the latent codes of real images, conditioned on label \(y\)).

The solution yields a stochastic differential equation (SDE) of the form

\[ dz_t = f(z_t, t, y)\,dt + g(t)\,dw_t, \]

where \(f\) is the drift we aim to learn, \(g\) is the (fixed) diffusion coefficient, and \(w_t\) is a Wiener process.
In this implementation we work with the probability flow ODE (the deterministic counterpart), which shares the same marginal distributions:

\[ \frac{dz_t}{dt} = f(z_t, t, y). \]

We learn \(f\) via a neural network (the drift network) and integrate the ODE from \(t=0\) to \(t=1\) to transform noise into a latent code that decodes to an image.

1.2 Latent Space Representation

Images \(\mathbf{x} \in \mathbb{R}^{3\times S\times S}\) (where \(S = \texttt{IMG\_SIZE}\)) are mapped to a spatial latent space \(\mathbf{z} \in \mathbb{R}^{C\times \frac{S}{8}\times \frac{S}{8}}\) (where \(C = \texttt{LATENT\_CHANNELS}\)) using a conditional VAE:

  • Encoder \(q_\phi(\mathbf{z}|\mathbf{x},y)\) outputs mean \(\boldsymbol{\mu}_\phi(\mathbf{x},y)\) and log‑variance \(\log\boldsymbol{\sigma}^2_\phi(\mathbf{x},y)\), both of shape \((C, \frac{S}{8}, \frac{S}{8})\).
  • Decoder \(p_\theta(\mathbf{x}|\mathbf{z},y)\) reconstructs the image.

The VAE is trained with the Evidence Lower Bound (ELBO):

\[ \log p(\mathbf{x}|y) \ge \mathbb{E}_{q_\phi}\big[\log p_\theta(\mathbf{x}|\mathbf{z},y)\big] \;-\; \beta\; \mathrm{KL}\big(q_\phi(\mathbf{z}|\mathbf{x},y)\;\|\; p(\mathbf{z})\big), \]

where \(p(\mathbf{z}) = \mathcal{N}(0,I)\) is the prior.
Because the latent is spatial, the KL divergence is summed over all \(C\times H\times W\) dimensions:

\[ \mathrm{KL} = -\frac{1}{2}\sum_{c,h,w}\Big(1 + \log\sigma_{c,h,w}^2 - \mu_{c,h,w}^2 - \sigma_{c,h,w}^2\Big). \]

To prevent latent collapse, we use a free bits trick: after computing the per‑sample KL, we clamp it to a minimum value FREE_BITS per sample.
We also add a diversity loss (see Phase 1) that encourages all latent channels to be active.

1.3 Ornstein‑Uhlenbeck Reference Bridge

Instead of the simple linear interpolation \(z_t = (1-t)z_0 + t z_1\), we can use the exact Ornstein‑Uhlenbeck bridge. The OU process is defined by the SDE

\[ dz_t = -\theta z_t\,dt + \sigma\,dw_t, \]

with stationary distribution \(\mathcal{N}\!\left(0, \frac{\sigma^2}{2\theta}\right)\).
Given two endpoints \(z_0\) (at \(t=0\)) and \(z_1\) (at \(t=1\)), the bridge at time \(t\) is Gaussian with

\[ \begin{aligned} \mathbb{E}[z_t|z_0,z_1] &= \frac{\sinh(\theta(1-t))}{\sinh\theta}\,z_0 \;+\; \frac{\sinh(\theta t)}{\sinh\theta}\,z_1, \\ \mathrm{Var}[z_t|z_0,z_1] &= \frac{\sigma^2}{2\theta}\; \frac{\sinh(\theta t)\,\sinh(\theta(1-t))}{\sinh\theta}. \end{aligned} \]

In the code, these formulas are implemented in OUReference.bridge_sample using exponential forms for numerical stability.
If USE_OU_BRIDGE = False, the simple linear interpolation is used.

1.4 Drift Network and Losses

The drift network \(f_\psi(z_t, t, y)\) learns the velocity that drives the probability flow.
The target is \(z_1 - z_0\) (for linear interpolation) or, with the OU bridge, the residual from the OU mean.

The drift loss is a time‑weighted Huber loss:

\[ \mathcal{L}_{\text{drift}} = \mathbb{E}_{t,z_0,z_1}\!\left[ \Big\|\, w(t)\big( \underbrace{(z_1 - z_0) - \mu_{\text{OU}}(t)}_{\text{target}} - f_\psi(z_t,t,y) \big) \Big\|_{\text{Huber}} \right], \]

where \(w(t) = 1 + \alpha\, t\) (with \(\alpha = \texttt{TIME\_WEIGHT\_FACTOR}\)) gives higher weight to later times, and the Huber loss uses \(\delta = 1\).

Additionally, a consistency loss keeps the current encoder close to a frozen reference encoder (from Phase 1) to stabilise training:

\[ \mathcal{L}_{\text{consistency}} = \mathrm{MSE}(\boldsymbol{\mu}_\phi, \boldsymbol{\mu}_{\text{ref}}). \]

2.3 PercentileRescale

This adaptive layer normalises activations channel‑wise to \([-1,1]\) based on running estimates of the 1st and 99th percentiles. During training, it updates the percentiles via exponential moving average. During inference or export, it uses the stored percentiles to scale:

\[ \text{out} = \tanh\!\left(0.9\,\frac{x - \text{low}}{\text{high} - \text{low}}\right). \]

The factor 0.9 provides a safety margin.

3. Training Phases and Loss Functions

Training is split into up to three phases (configurable). All losses are computed per batch and then averaged.

3.1 Phase 1 – VAE Training

Only the VAE (encoder + decoder) is trained. The drift network is frozen.

Loss components:

  1. Reconstruction loss – a weighted sum of
    • MSE (raw pixel error)
    • SSIM loss \(= 1 - \text{SSIM}(\text{recon}, \text{target})\) (implemented with a Gaussian window)
    • Perceptual loss – L2 distance on features from a frozen VGG‑16 (first 16 layers) after mapping images to \([0,1]\) and applying ImageNet normalisation.
    \[ \mathcal{L}_{\text{recon}} = \lambda_{\text{MSE}}\,\mathrm{MSE} \;+\; \lambda_{\text{SSIM}}\,(1-\mathrm{SSIM}) \;+\; \lambda_{\text{perc}}\,\mathrm{MSE}_{\text{VGG}}. \] Default weights: \(\lambda_{\text{MSE}}=2.5\), \(\lambda_{\text{SSIM}}=0.3\), \(\lambda_{\text{perc}}=0.2\times 0.2 = 0.04\) (the perceptual loss is further scaled by 0.2 internally).
  2. KL divergence – summed over spatial dimensions and free‑bits clipped: \[ \mathcal{L}_{\text{KL}} = \beta \; \frac{1}{B}\sum_{b=1}^B \max\!\left(\mathrm{KL}_b,\; \texttt{FREE\_BITS}\right), \] where \(\beta = \texttt{KL\_WEIGHT}\) (default \(10^{-4}\)) and \(\mathrm{KL}_b\) is computed per sample.
    An annealing factor (kl_annealing) linearly increases \(\beta\) over KL_ANNEALING_EPOCHS.
  3. Diversity loss – encourages all latent channels to be used and prevents collapse.
    For each batch we compute the channel‑wise standard deviation of \(\mu\): \(\sigma_c = \mathrm{std}_{B\times H\times W}(\mu[:,c,:,:])\). Then \[ \mathcal{L}_{\text{diversity}} = \gamma_{\text{low}}\,\mathbb{E}\!\left[\mathrm{ReLU}(\tau - \sigma_c)\right] \;+\; \gamma_{\text{high}}\,\mathbb{E}\!\left[\mathrm{ReLU}(\sigma_c - \sigma_{\max})\right] \;+\; \gamma_{\text{balance}}\,\mathrm{std}(\sigma_c), \] with \(\tau = \texttt{DIVERSITY\_TARGET\_STD}\) (adaptively annealed from _START to _END), \(\sigma_{\max} = \texttt{DIVERSITY\_MAX\_STD}\), and the \(\gamma\)s are taken from DIVERSITY_LOW_PENALTY, DIVERSITY_HIGH_PENALTY, DIVERSITY_BALANCE_WEIGHT.
    The term encourages channels to have similar, non‑zero, and not‑too‑large activity.

Total Phase 1 loss:

\[ \mathcal{L}_{\text{total}} = \mathcal{L}_{\text{recon}} + \mathcal{L}_{\text{KL}} + \texttt{DIVERSITY\_WEIGHT}\,\mathcal{L}_{\text{diversity}}. \]

3.2 Phase 2 – Drift Training

Now the drift network is trained while the decoder is frozen and the encoder is fine‑tuned (with a reduced learning rate). A frozen copy of the Phase 1 encoder (vae_ref) serves as an anchor for the consistency loss.

Loss components:

  1. Drift loss – time‑weighted Huber loss between the predicted drift \(f_\psi(z_t,t,y)\) and the target \(z_1 - z_0\) (or the OU‑adjusted target).
    Let \(w(t) = 1 + \alpha t\) (\(\alpha = \texttt{TIME\_WEIGHT\_FACTOR}\)). Then \[ \mathcal{L}_{\text{drift}} = \mathbb{E}\!\left[\,\mathrm{Huber}\!\left( w(t)\,(z_1 - z_0 - f_\psi(z_t,t,y)) ,\; \delta=1\right) \right] \cdot \texttt{DRIFT\_WEIGHT}. \] To avoid overfitting, the target is slightly noised with scale DRIFT_TARGET_NOISE_SCALE * (1‑t).
  2. Consistency loss – keeps the current encoder close to the reference: \[ \mathcal{L}_{\text{consistency}} = \mathrm{MSE}(\boldsymbol{\mu}_\phi, \boldsymbol{\mu}_{\text{ref}}). \] Its weight decays linearly from 1 to 0.1 over the remaining epochs of Phase 2: \[ \lambda_{\text{consistency}} = \texttt{CONSISTENCY\_WEIGHT} \times \max\!\left(0.1,\; 1 - \frac{\text{epoch} - \text{phase2\_start}}{\text{EPOCHS} - \text{phase2\_start}}\right). \]

Total Phase 2 loss:

\[ \mathcal{L}_{\text{total}} = \mathcal{L}_{\text{drift}} + \lambda_{\text{consistency}}\,\mathcal{L}_{\text{consistency}}. \]

3.3 Phase 3 – Joint Fine‑Tuning

Both encoder and decoder are unfrozen. The loss is the same as in Phase 2, but the learning rate for the VAE is further reduced (factor PHASE3_VAE_LR_FACTOR). The consistency loss still applies, with the same decay schedule.

4. Training Schedule

The schedule is controlled by the global dictionary config.TRAINING_SCHEDULE. Several modes are available:

  • auto – switch from Phase 1 to Phase 2 at a fixed epoch (switch_epoch, default 50).
  • three_phase – Phase 1 until switch_epoch_1, Phase 2 until switch_epoch_2, then Phase 3.
  • manual – force a specific phase (1,2,3) regardless of epoch.
  • custom – map epoch numbers to phases via a dictionary.
  • alternate – alternate between Phase 1 and Phase 2 every alternate_freq epochs.
  • vae_only / drift_only – always stay in Phase 1 or Phase 2.

The current phase is determined at the beginning of each epoch by set_training_phase(epoch). When a phase change occurs, the optimizer states are reconfigured (learning rates, frozen parameters) and a reference encoder is created for Phase 2.

5. Inference (Image Generation)

To generate an image conditioned on a label \(y\), we:

  1. Draw an initial latent \(z_0 \sim \mathcal{N}(0, \sigma_0^2 I)\) (by default \(\sigma_0 = 0.5\), or estimated from a real batch).
  2. Solve the ODE \(\frac{dz}{dt} = f_\psi(z,t,y)\) from \(t=0\) to \(t=1\) using a numerical integrator.
  3. Decode \(z_1\) with the VAE decoder to obtain the image.

5.1 ODE Integration

Three methods are implemented (configurable via method):

  • Euler: \(z_{t+\Delta t} = z_t + f(z_t,t,y)\,\Delta t\).
  • Heun (2nd order):
    \(k_1 = f(z_t,t,y)\),
    \(k_2 = f(z_t + \Delta t\,k_1, t+\Delta t, y)\),
    \(z_{t+\Delta t} = z_t + \frac{\Delta t}{2}(k_1 + k_2)\).
  • RK4 (4th order): standard Runge‑Kutta.

Time step \(\Delta t = 1 / \texttt{DEFAULT\_STEPS}\) (default 50). The latent is clamped to \([-10,10]\) after each step for stability.

5.2 Langevin Refinement

After ODE integration, an optional Langevin MCMC step can be applied to refine the latent (heuristic, not strictly derived from the SB). The drift at \(t=1\) is treated as a proxy for the score. For \(L\) steps:

\[ z \leftarrow z + \frac{\eta}{2}\, s\, f_\psi(z,1,y) + \sqrt{\eta}\,\epsilon,\qquad \epsilon\sim\mathcal{N}(0,I), \]

where \(\eta\) is langevin_step_size, \(s\) is langevin_score_scale, and the noise scale is annealed: \(\sqrt{\eta} \leftarrow \sqrt{\eta}\,(1-0.5\frac{\text{step}}{L})\). The latent is clamped to \([-2\sigma_{\max}, 2\sigma_{\max}]\) to stay within a reasonable range.

6. Key Features

  • Flexible Data Loading: support for multiple datasets (STL-10, CIFAR-10) or custom local image folders with automatic label discovery.
  • Configurable Resolutions: independent control over model input size (IMG_SIZE) and final generation size (GEN_SIZE).
  • MCP Design Pattern: strict separation of concerns between logic (Processor), state (Context), and interface (Display).
  • Label conditioning via Attention-augmented Residual blocks and scale‑shift modulation in every block.
  • Percentile rescaling – adaptive normalisation based on running percentiles, improving stability.
  • KPI tracking – real‑time monitoring of loss, SNR, latent statistics, with convergence detection and early stopping.
  • Snapshot ensemble – automatic saving of model snapshots every SNAPSHOT_INTERVAL epochs; emergency revert on NaN/Inf.
  • Ornstein‑Uhlenbeck bridge – exact OU bridge sampling for theoretically grounded training.
  • Multi‑device support – CUDA, MPS (Apple Silicon), DirectML, CPU.
